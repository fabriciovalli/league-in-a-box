Navigation Model (No Pill Tabs)

Replace tab-switching references with a product-grade navigation system: global marketing nav + role-aware app shell.

Navigation Requirements
Marketing navigation

Desktop: top rail with persistent CTA

  • Primary nav: Product, Solutions, Pricing, Case Studies, FAQ, Pilot Program.
  • Right actions always visible: Log in + Start a league.
  • Product opens a two-column mega menu for "League in a Box" and "Club Intelligence".
  • Mobile collapses to menu drawer with sticky "Start a league" CTA.
Application navigation

Desktop: left rail + top command bar

  • Left rail groups by workflow: Command, Fixtures, Results, Sponsors, Reports, Club Intelligence, Compliance.
  • Top command bar controls context: league selector, season selector, round/day, global search, quick actions.
  • User roles filter the left rail; users only see pages they can access.
  • Mobile uses bottom nav for top 4 destinations plus a "More" drawer.
Unique interaction pattern

Touchline Command Palette (press "/")

A command bar available across app pages to jump to any route, open a workflow, or trigger high-frequency actions (create fixture, resolve dispute, send reminders). This is the differentiator replacing tab-heavy navigation and supports volunteer time pressure.

App Pages - Specific Requirements

Page-level requirements for app implementation. Every item below is mandatory for v1 behavior unless explicitly marked as phase 2.

Core App Specs

Organiser Home

/app/organiser/home
Must includeKPI strip, pending result queue, dispute queue, pitch conflict alerts, sponsor delivery alert, deadline widget.
BehaviorAuto-refresh every 30s; optimistic updates for status changes; one-click drill-down to results and scheduling.
AcceptanceAt least 90% of unresolved results surfaced without scrolling on desktop.

League Control

/app/organiser/league-control
Must includeDrag-drop fixture board, pitch occupancy map, referee assignment panel, weather notice panel.
BehaviorConflict detection under 100ms after move; block save on unresolved overlap; auto-suggest two alternatives.
AcceptanceAll fixture edits produce audit entries with user, timestamp, and old/new values.

Result Inbox

/app/organiser/results
Must includeSubmission timeline, captain-vs-opponent comparison, dispute state machine, communication shortcuts.
BehaviorStates: submitted, awaiting confirmation, disputed, resolved, locked. Dispute notes mandatory before resolve.
AcceptanceMedian dispute resolution under 4 hours from first dispute flag.

Captain Result Link

/r/:token
Must includeSingle match context, goals and points input, computed totals preview, confirmation and success receipt.
BehaviorToken scoped to one match; token expiry enforced; duplicate submission idempotent with clear message.
AcceptanceMedian completion time under 30 seconds on mobile 4G conditions.

Public League Page

/league/:slug
Must includeStandings table, upcoming fixtures, recent results, sponsor slot, share controls, clear last-updated stamp.
BehaviorNo auth required; CDN cached with rapid invalidation after result confirmation; OG tags for sharing.
AcceptanceLargest contentful paint under 2.5s on median mobile.

Coach Hub

/app/club/:clubId/coach-hub
Must includeSquad status board, retention radar, attendance signals, upcoming match prep panel, action list.
BehaviorHighlight risk changes week-over-week; never rank minors publicly; support coach note capture in-line.
AcceptanceAt-risk players always surfaced in top section without additional filtering.

Squad Roster

/app/club/:clubId/squad/:squadId
Must includeRoster list, position and availability filters, quick-contact actions, attendance trend markers.
BehaviorBulk updates for training attendance; instant search by player and guardian names.
AcceptanceSearch returns first result in under 200ms with 300-player dataset.

Player Profile

/app/club/:clubId/player/:playerId
Must includeSeason timeline, appearance history, match context, coach notes, locked ratings/trial panel when consent missing.
BehaviorConsent gate checked on every load; lock state is explicit with reason and request-consent action.
AcceptanceNo ratings UI data is rendered when consent is absent or revoked.

Consent Vault

/app/club/:clubId/compliance/consent
Must includePer-player feature matrix, guardian status, request history, immutable audit timeline, reminder workflow.
BehaviorEvery consent change logs actor, channel, timestamp, and legal text version; exportable audit report.
AcceptanceAudit records cannot be edited through UI by any role.

Sponsor Studio

/app/organiser/sponsors
Must includeSlot inventory, campaign builder, promised-vs-delivered tracker, auto-generated sponsor report.
BehaviorWarn when campaign drops below contracted delivery threshold; suggest replacement placements.
AcceptanceWeekly sponsor report generated in one click and exportable to PDF.

Reports and Exports

/app/organiser/reports
Must includeCounty board standings pack, disciplinary report, sponsor campaign report, club intelligence summary export.
BehaviorScheduled exports with email delivery; manual on-demand export with progress indicator.
AcceptanceExports include generation timestamp and source season metadata.
